开启 AI 观察后，下班鸭会把屏幕里的工作信息和应用、时间对应起来，自动整理成当天的记录。你可以回看时间线、和小黄鸭继续聊，最后再把确认过的内容写成报告。

> 目前提供 Windows 桌面版。AI 识别和报告生成使用你自行配置的兼容 API 服务商；下班鸭不提供项目方账号或数据服务器。

## ✨ Why 下班鸭？

忙了一天后，很容易只记得自己很忙，却说不清时间都花在哪里。下班鸭在你开启 AI 观察后，会自动把屏幕里的工作信息、应用和时间整理起来。你可以在 Today 和 Timeline 里看明白今天或这一周做了什么，把零碎的时间重新拿回来。

另一边，小黄鸭一直在桌面上陪你上班。它知道你今天做了什么，也会理解你遇到的困惑；聊天时不需要反复交代背景，它会带着当天的记录和相关记忆陪你继续聊。遇到看不懂的内容，还可以直接框选问它。

## 👋 面向谁

适合以 Windows 电脑工作为核心的个人开发者、科研工作者等。

## 🖼️ 界面预览

### ☀️ 今天发生了什么

首页把当天的 AI 观察、当前状态、时间脉络和待确认事项放在一起。先看全貌，再进入具体证据。

![Today：当天工作概览](docs/images/today.png)

### 🔎 AI 观察是可回看的记录

每次识别都保留为可筛选的记录：包含时间、分类、应用和可确认的画面事实。它是后续时间线与报告的证据来源，而不是一段不可追溯的总结。

![AI 识别记录](docs/images/ai-observations.png)

### 🗺️ 用时间线还原工作脉络

周视图会将相近活动合并为连续片段。你可以快速看出专注、沟通、个人活动和离开电脑的分布，而不必翻找零散截图。

![Timeline：周视图概览](docs/images/timeline-week-overview.png)

需要更细时，可以放大查看同一周内的活动密度。

![Timeline：活动密度](docs/images/timeline-week-detail.png)

点开时间片段后，右侧会展示对应的观察事实、AI 推断、置信度和原始证据，方便判断这段记录是否可信。

![Timeline：片段证据详情](docs/images/timeline-week-inspector.png)

### 🐤 小黄鸭聊天：随时回到当下

桌宠是一个不打断工作的轻量入口。聊天时，小黄鸭可以结合当天的 AI 观察、窗口活动、空闲时段、最近对话和与问题相关的长期记忆，帮你回顾正在做什么、接着处理什么；它不会把不确定的推断伪装成事实。

![桌宠聊天入口](docs/images/desk-pet-chat.png)

### 👀 看图问鸭：框选屏幕，直接问

遇到看不懂的报错、代码、网页内容或文档时，从桌宠聊天发起“看图问鸭”：它会隐藏聊天窗并截取当前屏幕，你框选需要关注的区域、写下问题，随后在聊天中结合整屏上下文与框选细节回答。提问完成后立刻回到桌面，不必手动保存或转发截图。

![看图问鸭：框选屏幕并在聊天中继续回答](docs/images/ask-duck.gif)

### 🧠 Memory：让对话有连续性，也保留你能检查的依据

聊天里值得留下的决定、进展和偏好会整理成**事件卡**；项目、人物、工具和概念则形成**元素卡**，记录它们当前的状态和变化。平时小黄鸭只带着这些记忆的简短索引，需要时才按问题找回相关事件、状态变化和关键原话。真正被用到的记忆会保留得更久，不相关的会慢慢淡出；你可以查看来源、编辑、固定或遗忘它们。

![Memory 记忆系统：事件卡、元素卡与可检查的来源](docs/images/memory-system.png)

## 🧭 它怎样工作

1. **多模态 AI 观察**：你开启 AI 后，模型以屏幕画面为主，结合前台应用、时间和近期活动上下文，识别正在进行的工作与可观察的画面事实，并标出不确定性。
2. **形成工作记忆**：观察结果、活动时段和空闲信息保存在本机，按时间组织为可追溯的证据，而不是一串窗口标题。
3. **回看与确认**：Today、识别记录和 Timeline 让你检查一天发生了什么；必要时再补充、修正或删除。
4. **持续协作**：小黄鸭聊天会按需使用当天上下文和相关记忆卡，帮助你续上当前任务，而非把所有历史一股脑塞进对话。
5. **形成输出**：基于已确认的记录生成 Markdown 日报、周报或月报，之后仍可自行编辑。

## ✨ 功能

### 📥 Capture · 采集

你开启 AI 观察后，下班鸭以屏幕画面为主，结合前台应用、窗口标题、时间和空闲等上下文进行多模态识别，形成带时间、证据与置信度的观察记录。本地活动记录负责补足时间上下文，而不是代替理解工作内容。

### 🔍 Review · 回看

从 Today 开始总览当天状态，再进入 AI 识别记录和 Timeline 核对具体片段、置信度与原始证据。

### 🧠 Remember · 记住

小黄鸭聊天会结合当天工作上下文和与问题相关的记忆卡，帮助你续上对话、回顾进展与未确认的问题；你也可以从桌宠发起“看图问鸭”，框选当前屏幕直接提问。

### ✍️ Write · 写成稿

基于已确认的工作记录生成 Markdown 日报、周报或月报；生成后仍由你继续编辑。

### 🔒 Privacy · 隐私

本地 SQLite 保存数据；需要 AI 时，只向你自行配置的兼容 API 服务商发送必要的截图或文本上下文。

## 🔒 隐私与 AI 边界

- 下班鸭没有内置云端账号系统，也不会把数据发送给项目作者服务器。
- 本地活动、时间线、聊天记录和数据库默认保存在你的电脑上。
- 当你启用 AI 截图识别、聊天或报告生成时，相关截图、识别摘要或报告素材会发送到你在设置中自行配置的兼容 API 服务商。主动启动“看图问鸭”时，整屏截图用于理解上下文；提交问题后，框选区域截图也会用于回答。
- 你可以在生成报告前回看记录，也可以继续编辑生成的 Markdown 内容。

更多细节请见 [PRIVACY.md](PRIVACY.md)。

## ❓ FAQ

### 数据在哪里？

活动记录、时间线、聊天记录和报告默认保存在本机 SQLite 数据库中。下班鸭没有项目方账号系统或云端数据服务器。

### 不配置 API 能用什么？

非常建议配置 API，以体验 AI 观察、小黄鸭聊天、看图问鸭和报告生成等完整功能。不配置时，仍可使用本地前台活动追踪、空闲检测、原始记录、时间线和数据导出。

### API 花费大概多少？

按默认每 5 分钟一次的 AI 观察、一个正常工作日估算，多模态模型约为 ¥0.4。实际金额会随工作时长、截图间隔、所选模型和服务商变化。小黄鸭聊天和看图问鸭的花费主要取决于聊天频次，以及发送的文字和图片量。

### AI 会上传什么？

只有在你启用 AI 截图识别、聊天或报告生成时，相关截图、识别摘要或必要的文本上下文才会发送到你在设置中配置的服务商。主动启动“看图问鸭”时，整屏截图会用于理解上下文；提交问题后，框选区域截图也会一并用于回答。项目作者不会接收这些数据。

### 如何停止记录、导出或删除数据？

你可以在首页停止 Vision Auto，并在设置中关闭自动追踪；在“数据管理”中可导出 JSON，也可以清空本地数据。清空前请先导出需要保留的内容。

### 当前支持什么系统？

当前提供 Windows 桌面版。

## 🚀 安装

### Windows 用户

前往 [GitHub Releases](https://github.com/diqierjia/xiabanya-public/releases/latest) 下载最新 Windows 安装包，安装后在设置中配置你的 AI 服务商和 API Key（如果需要 AI 识别或报告生成）。

### 从源码运行

环境要求：Node.js 与 npm。

```bash
npm install
npm run dev
```

构建 Windows 安装包：

```bash
npm run dist:win
```

运行测试：

```bash
npm test
```
