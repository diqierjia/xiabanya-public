# Privacy

下班鸭是本地优先的桌面应用。本文说明默认情况下哪些数据保存在本机，哪些数据会在用户启用 AI 功能时发送到外部服务。

## 本地保存的数据

应用会在本机 SQLite 数据库中保存：

- 前台应用名、窗口标题、活动开始和结束时间
- 手动记录、分类、备注和报告内容
- AI 识别结果的标题、分类、摘要和原始响应
- 用户设置，包括 SiliconFlow API Key

开发模式下数据库位于项目 `data/` 目录；打包后位于 Electron 的系统 `userData` 目录。公开仓库不包含任何用户数据库、日志、截图或 debug 文件。

## 发送到外部服务的数据

只有在用户配置 API Key 并启用 AI 功能后，应用才会调用 SiliconFlow 兼容接口：

- AI 截屏识别会发送当前屏幕截图、前台应用名和窗口标题
- AI 报告生成会发送所选日期范围内的识别摘要和活动记录
- API Key 只用于请求用户配置的 AI 服务商

项目作者不运营中转服务器，也不会接收这些请求内容。外部服务的数据处理规则以用户所配置服务商的条款为准。

## 截图处理

当前实现将截图作为内存中的请求内容发送给 AI 服务商，不在仓库或发布包中预置任何截图数据。应用不会把完整报告 prompt 写入 debug 文件。

## 用户控制

用户可以：

- 不配置 API Key，仅使用本地记录能力
- 关闭自动 AI 截屏识别
- 在设置中导出、导入或清空本地数据
- 删除历史报告和识别结果
